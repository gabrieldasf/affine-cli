# Requirements Archive: v1.1 Canvas Power Tools

**Status:** shipped 2026-06-19
**Audit:** `.planning/v1.1-MILESTONE-AUDIT.md`

## Completed Requirements

- [x] **CANVAS-01: Canonical Canvas Snapshot Model**

  Outcome: satisfied. Search, diff, and transform share the canonical selector model for Canvas blocks, notes/cards, connectors, text, geometry, display mode, image/source metadata, hierarchy, and integrity signals.

- [x] **CANVAS-02: Block-Level Search And Selection**

  Outcome: satisfied. `canvas search` supports text, exact ID, flavour, type, display mode, geometry bounds, connector endpoints, snapshot source, and history timestamp, with reusable JSON selector output.

- [x] **CANVAS-03: Semantic Canvas Diff**

  Outcome: satisfied. `canvas diff` compares Canvas states and reports semantic changes including added, removed, text-changed, geometry-changed, display-mode-changed, relinked, image/source-changed, orphaned, unreachable, and integrity-changed entities.

- [x] **CANVAS-04: Machine-Applyable Transform Plans**

  Outcome: satisfied. `canvas transform` produces dry-run operation plans for move, resize, align, distribute, display mode, and selected metadata updates with affected IDs, before/after values, integrity, validation, rollback, and proof fields.

- [x] **CANVAS-05: Gated Live Apply**

  Outcome: satisfied with warning. `canvas apply --live` requires explicit apply flags, workspace/doc targeting, backup directory, confirmation, backup-before, delta artifact, pre/post integrity, and reload verification. Exact post-field equality assertion remains future hardening.

- [x] **CANVAS-06: Printing Press Durability**

  Outcome: satisfied. Hand-authored Canvas changes are represented in patch records, and README, SKILL, `which`, command help, workflow proof, and publish validation are aligned.

- [x] **CANVAS-07: Workflow Verification**

  Outcome: satisfied with warning. `workflow-verify` passes and live proof exists. The durable workflow manifest should still add search, diff, and doc-integrity smoke steps.

## Non-Goals Preserved

- Did not rewrite AFFiNE renderer or Blocksuite internals.
- Did not enable broad live mutation before search, diff, and transform contracts stabilized.
- Did not hardcode Quartzo workspace, document, or block IDs into reusable code.
- Did not treat generated GraphQL search endpoints as Canvas block-level search.

- `cli-printing-press publish validate --dir library/affine --json` passes.
