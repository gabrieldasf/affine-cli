# Milestone v1.1: Canvas Power Tools

**Status:** shipped 2026-06-19
**Phases:** 1-4
**Total Plans:** 4
**Audit:** `.planning/v1.1-MILESTONE-AUDIT.md`

## Overview

v1.1 turned AFFiNE Canvas support from diagnostics into an operator workflow:

1. Search and select exact Canvas entities.
2. Diff expected, historical, snapshot, and live states.
3. Produce machine-applyable dry-run transform plans.
4. Apply live changes only through explicit, verified, backed-up gates.

## Phases

### Phase 1: Canvas Search And Selection

**Goal:** Create a read-only canonical Canvas snapshot model and expose `canvas search` for block/card/connector-level selection.

**Requirements covered:** CANVAS-01, CANVAS-02, CANVAS-06 partial.

**Evidence:** `.planning/phases/01-canvas-search-and-selection/VERIFICATION.md`

Result: `canvas search` emits stable selector JSON across live, snapshot, and history sources, and local snapshot search no longer requires live workspace/doc flags.

### Phase 2: Canvas Diff And History Diagnosis

**Goal:** Add `canvas diff` to compare two canonical Canvas states and diagnose missing, changed, moved, relinked, orphaned, or integrity-damaged entities.

**Requirements covered:** CANVAS-01, CANVAS-03, CANVAS-07 partial.

**Evidence:** `.planning/phases/02-canvas-diff-and-history-diagnosis/VERIFICATION.md`

Result: `canvas diff` reports stable semantic categories across snapshot, history, and live comparison modes without adding mutation paths.

### Phase 3: Canvas Transform Dry-Run Plans

**Goal:** Add `canvas transform` as a dry-run planner that consumes selectors and emits machine-applyable operation plans.

**Requirements covered:** CANVAS-04, CANVAS-06 partial, CANVAS-07 partial.

**Evidence:** `.planning/phases/03-canvas-transform-dry-run-plans/VERIFICATION.md`

Result: transform plans support move, resize, align, distribute, display mode, and selected metadata changes, and are accepted by `canvas apply --dry-run`.

### Phase 4: Gated Live Apply And Publish Proof

**Goal:** Enable live Canvas apply for validated operation plans and prove the final CLI is safe, documented, and publishable.

**Requirements covered:** CANVAS-05, CANVAS-06, CANVAS-07.

**Evidence:** `.planning/phases/04-gated-live-apply-and-publish-proof/VERIFICATION.md`

Result: `canvas apply --live` supports transform and layout plans behind explicit `--workspace`, `--doc`, `--backup-dir`, and `--yes` gates, with backups, delta capture, pre/post integrity checks, reload verification, workflow proof, publish validation, and approved live AFFiNE smoke.

## Milestone Summary

**Key accomplishments:**

- Added reusable Canvas search selectors for blocks, cards, connectors, text, geometry, display mode, source mode, and history/snapshot sources.
- Added semantic Canvas diff categories for expected vs actual state diagnosis.
- Added machine-applyable dry-run transform plans with deterministic IDs and proof fields.
- Added gated live apply for transform and layout plans with backup-before, delta artifacts, integrity checks, reload verification, and explicit confirmation.
- Proved the final workflow through tests, workflow verification, publish validation, and a live AFFiNE smoke that created 20 Live Zap cards and 28 connectors.

**Known tech debt accepted at close:**

- Durable workflow proof omits search, diff, and doc-integrity smoke steps.
- Live transform apply does not yet assert every reloaded field equals every `TransformOperation.After` value.
- Phase directories have `VERIFICATION.md` but no `SUMMARY.md` frontmatter.
- Nyquist `VALIDATION.md` artifacts are missing.
- v1.1 requirements did not use a formal traceability table.

For current project status, see `.planning/PROJECT.md`.
